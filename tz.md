# Техническое задание и Анализ проекта GachiBass Radio

## Обзор проекта
Проект представляет собой одностраничный сайт (SPA), посвященный Gachi-культуре. Сайт объединяет в себе радио-плеер, ссылки на YouTube-каналы и аудио-галерею.

## Структура файлов
- `index.html`: Основная разметка страницы.
- `css/style.css`: Стили оформления (темная тема, золотые акценты).
- `js/content.js`: Логика загрузки контента из JSON и динамического построения интерфейса.
- `content.json`: База данных контента (в данный момент содержит список аудио-треков).
- `assets/`:
    - `audio/`: MP3 файлы треков Swallowsher MorGaySlave.
    - `description/`: Текстовые описания легенд гачи.
    - `pictures/`: Изображения легенд (webp, jpg).
    - `videos/`: Фоновое видео (`video_bg.mp4`).

## Текущий функционал
1. **Радио-плеер**: В шапке сайта встроен плеер, транслирующий поток `https://radio.gachibass.us.to/fisting`.
2. **Hero-секция**: Динамически создается через JS; включает фоновое видео и изображения легенд (Billy Herrington, Van Darkholme).
3. **Портал каналов**: Сетка ссылок на тематические YouTube-каналы с возможностью вставки iframe для определенных каналов.
4. **Аудио-галерея**: Загружается из `content.json`, отображает список треков с плеерами. Реализован эксклюзивный режим воспроизведения (при запуске одного трека остальные останавливаются).

## Замеченные несоответствия/ошибки
- В `js/content.js` (строки 21, 25) используются пути `assets/billy.webp` и `assets/van.jpg`, но в структуре папок изображения находятся в `assets/pictures/`.
- Файл `content.json` содержит пустую категорию `videos`.
- Есть файл `listFiles.txt` и `TZ_VIDEO.md`, назначение которых требует уточнения.
